# AGENTS.md

Scraper + processor for Indian Supreme Court judgments, published to the public S3 bucket `s3://indian-supreme-court-judgments` (AWS Open Data). See `README.md` for S3 layout, index schema, and parquet columns.

## Environment

- Python **3.13**, managed by **`uv`** — not pip. Setup: `uv sync && source .venv/bin/activate`.
- Add deps with `uv add <pkg>`. Don't hand-edit `requirements.txt`.
- Lint/format: `ruff check .` / `ruff format .` (also in `.pre-commit-config.yaml`).

## Verification

There is **no test suite**. For changes to the scraper, archive, sync, or metadata code: run the script against a single small date range or year and inspect the output. Do not claim correctness from lint or type checks alone.

## Hard rules

- **Be gentle to ecourts.gov.in.** Don't raise concurrency, shorten sleeps, or add aggressive retries in `download.py` without an explicit ask — this is an explicit project value in `README.md`.
- **The S3 bucket is production data.** Any path that writes to `s3://indian-supreme-court-judgments/...` publishes to the real dataset. Use `boto3` with `UNSIGNED` config for exploration; confirm before running upload paths.
- Archives are **uncompressed `.tar`** with the V2 index format. The zip-era code was removed — don't reintroduce it if you see references in git history.
- `prune/` holds one-shot migration/verification scripts. They are **not** part of normal runs; don't call them from other code.
- `dataset_sizes.csv` is regenerated by `calculate_dataset_sizes.py` — don't hand-edit.
- Skip scratch dirs when searching: `sc_data/`, `packages/`, `local_sc_judgments_data/`, `captcha-tmp/`, `captcha-failures/`, `temp-files/`. Some hold tens of thousands of files; scope `Glob`/`Grep` with explicit paths.
